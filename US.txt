# Оформление заказа

## Цель: оформить сформированный заказ

**Основной сценарий** (_пользователь авторизован_):

+ Пользователь нажимает на кнопку “оформить заказ”

+ Система переводит пользователя на страницу оформленного заказа, где формируется внешний вид заказа. 

+ Система проверяет, авторизован ли пользователь.

+ Система отображает выбранные пиццы, указывает изменения в составе, если таковые были, отображает поля персональных данных, заполненные информацией из личного кабинета пользователя.

+ Пользователь проверяет данные и подтверждает оформление заказа.

+ Систем проверяет корректность введенных данных (валидация). В случае обнаружения ошибок, система выводит информационное окно для пользователя с указанием некорректно введенных данных.

+ Заказ оформлен и сформирован.



**Альтернативный сценарий** (_пользователь не авторизован_):

+ Пользователь нажимает на кнопку “оформить заказ”

+ Система переводит пользователя на страницу оформленного заказа, где формируется внешний вид заказа. 

+ Система проверяет, авторизован ли пользователь.

+ Система отображает выбранные пиццы, указывает изменения в составе, если таковые были, отображает поля персональных данных, отображает пустые поля персональных данных, необходимых для оформления заказа.

+ Пользователь проверяет заказ, вводит затребованные данные и подтверждает оформление заказа.

+ Систем проверяет корректность введенных данных (валидация). В случае обнаружения ошибок, система выводит информационное окно для пользователя с указанием некорректно введенных данных.

+ Система отправляет код-подтверждение на указанный номер телефона пользователя и отображает форму для ввода кода-подтверждения.

+ Пользователь вводит код.

+ Система проверяет код.

+ Если код проверен успешно, то заказ оформлен и сформирован. В противном случае система отображает информационное окно, что код введен некорректно и запрашивает повторный ввод кода.


*А тут у нас будут изменения для Основной ветки, не содержащие изменение, созданные в новой*
